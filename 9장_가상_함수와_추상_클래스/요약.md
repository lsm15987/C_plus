## 함수의 재정의
- 업캐스팅이나 다운 캐스팅을 통해서 언제든 부모든 자식이든 멤버 함수를 실행할 수 있다.
## 재정의와 오버라이딩
- 재정의는 부모와 자식의 멤버 둘 다 사용한다.
- 오버라이딩은 virtual 함수를 이용하여 부모의 멤버 함수는 죽여버리고 자식 멤버 함수만 쓴다.
### 함수 오버라이딩
- 자식 클래스에서 부모 클래스의 가상 함수와 동일한 이름의 함수 선언
	- 부모 클래스의 가상 함수의 존재감 상실시킴
	- 자식 클래스에서 오버라이딩한 함수가 호출되도록 동적 바인딩
	- 함수의 재정의라고도 부름
- 오버라이딩의 목적
	- 파생 클래스에서 구현할 함수 인터페이스 제공
	- 다형성의 실현
- 성공 조건
	- 가상 함수 이름, 매개변수 타입과 개수, 리턴 타입이 모두 일치(접근 지정자는 자유롭게)
	- 오버라이딩 시 자식 클래스에서 virtual 지시어 생략 가능

### 가상 함수
- virtual 키워드로 선언됨 멤버 함수
- 동적 바인딩 지시어
- 컴파일러에게 함수에 대한 호출 바인딩을 실행 시간까지 미루도록 지시
```cpp
class Base{
public:
	virtual void f();
};
```
## 동적 바인딩
- 부모 클래스에 대한 포인터로 가상 함수를 호출하면,
- 객체 내에 오버라이딩한 자식 클래스 함수를 찾아 실행
- 이 과정이 실행 중에 이루어지기에 동적 바인딩이라고함
## 범위 지정 연산자(::)
- 정적 바인딩 지시
- 부모 클래스::가상 함수() 형태로 부모 클래스의 가상 함수를 정적 바인딩으로 호출
## 가상 소멸자
- 소멸자를 virtual 키워드로 선언
- 소멸자 호출 시 동적 바인딩 발생
- 자식 클래스의 소멸자가 실행되면 부모 클래스의 소멸자가 실행되도록 한다.
