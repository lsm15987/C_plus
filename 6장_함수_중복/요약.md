## 디폴트 매개 변수
- 매개 변수에 값이 넘어오지 않는 경우, 디폴트 값을 받도록 선언된 매개 변수
- 디폴트 매개 변수는 끝 쪽부터 초기값을 순서대로 선언되어야 한다.
- 이를 이용하여 함수를 간소화 할 수 있다.
	- 다형성을 이용하여 만들어진 생성자를 하나의 함수로 나타낼 수 있음.
## 함수 중복의 모호성
1. 형 변환으로 인한 모호성
	- 매개 변수의 형 변환으로 인한 중복 함수 호출의 모호성
	- 입력으로 int 값을 받았는데 함수 정의는 double 과 float 뿐. 그런다면 어떤 것을 선택해야하나
2. 참조 매개 변수로 인한 모호성
	- 함수의 입력 값이 함수 정의에서 매개 변수가 참조인지 값인지에 대하여 
3. 디폴트 매개 변수로 인한 모호성
	- 디폴트 매개 변수를 이용한건지 아니면 없는 것을 사용한건지
## static 멤버
### non-static 멤버
```cpp
class Sample{
	int n;
	void f();
};
```
- 객체마다 별도 생성
- 객체와 생명을 같이함
- 객체 별로 공유 되지 않음
### static 멤버
```cpp
class Sample{
	static int n;
	static void f();
};
```
- 클래스 당 하나 생성
- 프로그램과 생명을 같이 함
- 동일한 클래스의 모든 객체들에 의해 공유됨
- 같은 클래스끼리 공유하는 멤버
- 접근은 객체.멤버
- 다른 접근으로 클래스::멤버
- 클래스 선언 때 static 타입 멤버
- static 멤버 변수이면 전역 공간에 초기화 해줘야함
### static의 주요 활용
- 전역 변수나 전역 함수를 클래스에 캡슐화
- 객체 사이에 공유 변수를 만들고자 할 때
### static 멤버 함수의 접근
- static 멤버 함수가 접근할 수 있는것
	- static 멤버 함수
	- static 멤버 변수
	- 함수 내의 지역 변수
- static 멤버 함수는 non-static 멤버에 접근 불가능
	- 왜냐하면 객체가 생성되지 않은 시점에서 static 멤버 함수가 호출될 수 있기 때문
- 하지만 반대로 non-static 멤버는 static 멤버에 접근이 가능하다
- static 멤버는 객체 생성 전에 클래스::static 멤버 로 접근이 가능
- static 멤버 함수 에서 this 사용 불가능
