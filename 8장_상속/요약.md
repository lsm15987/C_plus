## 상속
- 클래스 사이에서 상속관계 정의(객체 사이 x)
- 기본 클래스(부모 클래스)
- 파생 클래스(자식 클래스)
- 기본 클래스에서 파생 클래스로 갈수록 클래스의 개념이 구체화
- 다중 상속을 통한 클래스의 재활용성 높임
### 상속의 목적 및 장점
- 간결한 클래스 작성
- 클래스 간의 계층적 분류 및 관리의 용이함
- 클래스의 재사용과 확장을 통한 소프트웨어 생산성 향상
### 상속 선언
```cpp
class Student : public Person{
	...
}
```
- 클래스 선언할 때 : 이용해서 접근지정과 부모 클래스를 작성한다.
### 업 캐스팅
- 파생클래스 객체포인터를 기본클래스 객체 포인터로 치환
- 업 캐스팅을 통해 같은 멤버를 공유하는 객체를 만드는듯
### 다운 캐스팅
- 기본클래스의 객체포인터를 파생클래스의 객체포인터로 치환
- 다운 캐스팅도 같은 멤버를 공유하는 객체를 만든다.
- 하지만 접근 가능한게 부모까지인지 아니면 자식까지인지를 정한다.
### protected 접근 지정
- private 멤버
	- 선언된 클래스 내에서만 접근 가능
	- 파생 클래스에서도 기본 클래스의 private 멤버 직접 접근 불가
- public 멤버
	- 선언된 클래스나 외부 어떤 클래스, 모든 외부 함수에 접근 허용
	- 파생 클래스 또한 접근 가능
- protected 멤버
	- 선언된 클래스에서 접근 가능
	- 파생 클래스에서만 접근 허용
	- private하며 + 파생 클래스만 접근을 허용
### 상속 관계의 생성자와 소멸자
- 생성자
	- 자식 -> 부모 순서로 생성자를 호출하여
	- 부모 -> 자식 순서로 호출한 생성자를 실행한다.
	- 자식 생성자에서 부모 생성자 호출에 언급이 없으면 자동으로 기본 생성자를 호출한다.
- 소멸자
	- 자식 -> 부모 순서로 소멸자가 실행된다.(생성자의 역순)
### 상속 지정
- 상속 선언 시 public, private, protected의 3가지 중 하나 지정
- 기본 클래스의 멤버의 접근 속성을 어떻게 계승할지 지정
	- public
		- 기본 클래스의 protected, public 멤버 속성을 그대로 계승
	- private
		- 기본 클래스의 protected, public 멤버 속성을 private로 계승 
	- protected
		- 기본 클래스의 protected, public 멤버 속성을 protected로 계승
### 다중 상속
- 기존 상속과 같다
- 자식:접근 지정 부모, 접근 지정 부모.. 
### 다중 상속의 문제점
- 기본 클래스 멤버의 중복 상속
- 같은 부모를 상속 받은 다른 자식 클래스를 동시에 상속 받는 손자 클래스가 같은 부모를 가져 중북 상속 발생
- 이를 해결하기 위한 가상 상속
### 가상 상속
- 자식 클래스의 선언문에서 기본 클래스 앞에 virtual 키워드를 선언
- 자식 클래스의 객체가 생성될 때 부모 클래스의 멤버는 오직 한번만 생성
	```cpp
	class In:virtual public BaseIO{
		...
	}
	```
